# 공수비교표 셀별 산출 공식

위치: **Project Management > View > 기본정보 섹션 하단 "프로젝트 공수 예상/발생 비교표"**

---

## 0. 기간(Period) 처리 — 가장 중요

표 제목: **"프로젝트 공수 예상/발생 비교표(2026.01.01)"**
화면 상단 필터: **시작 연월(yearMonthFrom) ~ 종료 연월(yearMonthTo)** (예: 2026.04 ~ 2026.05)

### 0-A. 표 제목의 "(2026.01.01)" 의미
- 해당 표가 "어느 시점 기준의 스냅샷인가" 또는 "프로젝트 시작일"을 나타내는 라벨로 보임
- **확인 필요**: 이 날짜가 (a) 프로젝트 시작일인지, (b) 표 산정 기준 시점(snapshot)인지, (c) 회계연도 시작일인지 명확히 해야 함

### 0-B. 화면 상단 yearMonthFrom~yearMonthTo 필터의 적용 범위
세 컬럼이 기간 필터에 반응하는 방식이 서로 다르다:

| 컬럼 | 기간 필터 적용 방식 |
|------|---------------------|
| **VRB 예상 공수** | Pre-Sales 입력은 일자 단위가 아닌 "프로젝트 전체 계획값" → 기간 필터로 잘라낼 수 없음. **항상 전체 계획 누적**으로 표시하거나, Pre-Sales 모달의 월별 분할 입력값 중 [yearMonthFrom..yearMonthTo] 범위만 합산 |
| **프로젝트 투입인력관리** | 각 인력의 [투입일자~철수일자] 구간이 [yearMonthFrom 1일 ~ yearMonthTo 말일] 과 겹치는 일수만 일할 계산하여 합산 |
| **개인별 Work Report** | Work Report 업무일지 기준일이 [yearMonthFrom 1일 ~ yearMonthTo 말일] 범위에 속하는 행만 합산 |

### 0-C. 본 문서 전반의 기간 변수 정의
아래 셀별 공식에서 사용하는 기간 변수:
```
P_START = yearMonthFrom 의 1일 (예: 2026-04-01)
P_END   = yearMonthTo 의 말일 (예: 2026-05-31)
```

## 컬럼별 데이터 출처 요약

- **VRB 예상 공수** ← Pre-Sales 단계 "매출/매입 계획 입력 모달"의 매입 > 내부인력 행
- **프로젝트 투입인력관리** ← Project 단계 "투입인력관리 모달"의 인력 행
- **개인별 Work Report** ← Daily Work Log "Work Report"의 업무내용 행

## 셀별 정확한 공식

### (1) 특급 × VRB 예상 공수
**옵션 A — 전체 누적(필터 무시):**
```
= SUM(
    Pre-Sales 매출/매입 계획 입력 모달의
    [구분=매입 AND 세부구분=내부인력 AND 등급=특급] 행들의 수량(M/D)
  )
```
**옵션 B — 기간 필터 적용(월별 분할 입력 사용 시):**
```
= SUM(
    Pre-Sales/Project 매출/매입 모달의
    [구분=매입 AND 세부구분=내부인력 AND 등급=특급] 행들의
    월별 금액 중 [P_START..P_END] 범위 월의 금액 합 ÷ 10,000,000
  )
```
**참조 파일:** `05-pre-sales-sales-cost-plan-initial-input-modal.html`
**단가 참조:** 특급 단가 10,000,000원
**확인 필요:** 옵션 A/B 중 어느 것이 시스템 정책인지

### (2) 특급 × 프로젝트 투입인력관리
```
= SUM(
    Project 투입인력관리 모달의
    [직책이 특급에 매핑되는 인력] 행들 각각에 대해
    OVERLAP_DAYS(투입일자, 철수일자, P_START, P_END) × 투입비율(%) ÷ 100
  )

OVERLAP_DAYS(a_start, a_end, p_start, p_end)
  = MAX(0, MIN(a_end, p_end) - MAX(a_start, p_start) + 1)
```
**참조 파일:** `09-project-deployed-employees-management-modal.html`
**직책→등급 매핑:** 이사↑ = 특급
**주의:** 기간 필터가 없으면 (철수일자 - 투입일자 + 1) × 투입비율 ÷ 100 전체

### (3) 특급 × 개인별 Work Report
```
= SUM(
    Work Report 업무내용 행들 중
    [업무구분=프로젝트
     AND 프로젝트명=현재프로젝트
     AND 작성자 직책=특급
     AND 업무일지 기준일(seqDt) BETWEEN P_START AND P_END]의
    소요시간(분) ÷ 480
  )
```
**참조 파일:** `15-work-report-view-edit.html`
**환산:** 1 M/D = 480분 (8시간 기준)
**작성자 직책:** Work Report의 사원아이디(userId)로 직원마스터 조회 → 직책 → 등급 매핑

---

### (4) 고급 × VRB 예상 공수
(1)과 동일 구조, 등급=고급 으로만 변경. 단가 9,000,000원.

### (5) 고급 × 프로젝트 투입인력관리
(2)와 동일 구조, 직책=부장 인력만 대상. 기간 필터(OVERLAP_DAYS) 동일 적용.

### (6) 고급 × 개인별 Work Report
(3)과 동일 구조, 작성자 직책=부장 조건만 변경. seqDt BETWEEN P_START AND P_END 동일 적용.

---

### (7) 중급 × VRB 예상 공수
(1)과 동일 구조, 등급=중급. 단가 7,000,000원.

### (8) 중급 × 프로젝트 투입인력관리
(2)와 동일 구조, 직책 IN (차장, 과장) 인력 대상. 기간 필터 동일.

### (9) 중급 × 개인별 Work Report
(3)과 동일 구조, 작성자 직책 IN (차장, 과장). 기간 필터 동일.

---

### (10) 초급 × VRB 예상 공수
(1)과 동일 구조, 등급=초급. 단가 5,000,000원.

### (11) 초급 × 프로젝트 투입인력관리
(2)와 동일 구조, 직책 IN (대리, 주임, 사원) 인력 대상. 기간 필터 동일.

### (12) 초급 × 개인별 Work Report
(3)과 동일 구조, 작성자 직책 IN (대리, 주임, 사원). 기간 필터 동일.

---

### (13) 합계 × VRB 예상 공수

```
= (1) + (4) + (7) + (10)
```


### (15) 합계 × 개인별 Work Report
```
= (3) + (6) + (9) + (12)
= 특급 WR + 고급 WR + 중급 WR + 초급 WR
```

## 중급 등급 예시 — 전체 기간 vs 기간 필터(2026.04~05) 비교

가정: 프로젝트 전체 기간 = 2026.03.01 ~ 2026.06.30

### 예시 (7) — 중급 × VRB 예상 공수
```
Pre-Sales 매입 계획(월별 분할): 03월=2, 04월=2, 05월=2, 06월=2 (단위 M/D)
- 전체 기간          → 2 + 2 + 2 + 2 = 8 M/D
- 04~05월만 선택 시  → 2 + 2         = 4 M/D
```

### 예시 (8) — 중급 × 프로젝트 투입인력관리
```
홍과장(중급) 투입 03.01~06.30, 비율 100% → 전체 122일 = 122 M/D
- 전체 기간          → 122 M/D
- 04~05월만 선택 시  → 04.01~05.31 겹침 61일 × 1.0 = 61 M/D
```

### 예시 (9) — 중급 × 개인별 Work Report
```
홍과장 03.20 480분 / 04.10 240분 / 05.05 120분 / 06.15 480분
환산 = 1.0 / 0.5 / 0.25 / 1.0 M/D
- 전체 기간          → 1.0 + 0.5 + 0.25 + 1.0 = 2.75 M/D
- 04~05월만 선택 시  →       0.5 + 0.25       = 0.75 M/D
```

| 등급 | 단가 |
|------|------|
| 특급 | 10,000,000원 |
| 고급 | 9,000,000원 |
| 중급 | 7,000,000원 |
| 초급 | 5,000,000원 |

## 참고: M/D 환산 기준

- 1 M/D = 1일 = 480분 (8시간)
- Work Report 소요시간(분) ÷ 480 = M/D
- 예: 120분 = 0.25 M/D, 240분 = 0.5 M/D, 480분 = 1.0 M/D

## 기간(Period) 관련 추가 의문점

1. **표 제목의 "(2026.01.01)" 정체** — 프로젝트 시작일? 표 산정 기준 시점? 회계연도 시작?
2. **VRB 예상 공수 컬럼이 기간 필터에 반응하는가** — 반응한다면 월별 분할 입력값(매출/매입 모달의 04월/05월 컬럼)을 기준으로 잘라내야 함
3. **투입인력관리 기간 겹침 처리** — 인력의 투입~철수 구간 일부만 [P_START..P_END]에 겹칠 때 일할 계산이 맞나, 아니면 전체 M/D를 다 잡나
4. **반올림/단위 일치** — 세 컬럼의 단위가 모두 M/D인데 산정 방식이 달라 소수점 처리 정책이 동일해야 함
5. **표 제목 날짜와 yearMonthFrom~To의 관계** — 둘 다 표시되는데 우선순위 또는 해석 규칙이 명확해야 함
6. **종료 연월의 "월 말일" 처리** — yearMonthTo=2026.05 → 2026-05-31 까지 포함, 윤년 처리 등 경계값 정의 필요
